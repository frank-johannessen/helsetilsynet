Rebuild links

The Rebuild links function is not often used, but you may need to rebuild links if the root directory has been changed or if you have replaced the page template that is linked to a specific page type.
The Update links for all pages option is very rarely used because other EPiServer functionality handles the problem. However, errors may occur that this function will correct. One scenario: You have a page type, such as a normal Web page, that is linked to the page template Page.aspx. You change it so that it is linked to page template Page2.aspx instead. Normally, EPiServer re-associates all pages created with the former page type, and they display using the new page template. There may be exceptions, though, where pages still display using Page.aspx. In this case, you will need to rebuild the links with the Update links for all pages function.

Use the function Specify old value of the root directory to be replaced if you have moved the root directory on the server. If you do not rebuild the links, the internal links for the whole Web site will be incorrect. To rebuild links, specify the old root directory and select OK.
